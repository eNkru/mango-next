# 主页继续阅读区布局优化

## Goal

将首页「继续阅读」从宽卡网格改为「首项大卡 + 紧凑竖列表」，显著降低垂直占用与信息密度，突出最近一本与阅读进度，并与「开始阅读」poster rail 保持形态差异。

## Confirmed Facts（代码）

- 入口：`frontend/src/pages/HomePage.tsx`；区块仅在 `continue_reading.length > 0` 时渲染。
- 当前布局：响应式宽卡网格 `.mango-continue-grid`（`minmax(320px)`），每卡左封面 + 右详情（标题、页码、进度条、继续/打开）。
- 数据：最多 8 条（API `limit = 8`；DB 取 20 再截断）；字段含 cover、name、page/pages、progress、title_id/id。
- 同页对照：`开始阅读` / `最近添加` 使用横向滚动 `PosterCard` rail。
- 继续阅读卡为内联 JSX；`ProgressBar` 可复用；空态仍为无数据时整块不渲染。

## User Intent

| 决策 | 结论 |
|------|------|
| 痛点 | 占高、信息过重、进度难扫 |
| 布局 | 首项大卡 + 其下紧凑竖列表 |
| 差异化 | 不用 start/recent 同款 poster rail |
| 大卡字段 | 封面、标题、粗进度条、页码、仅「继续」 |
| 列表行 | 小封面、标题、细进度条；整行进阅读器 |
| 条数/折叠 | API 仍 8；列表默认露 3 行，可展开 |

## Requirements

1. 将「继续阅读」改为 **焦点大卡 + 次要竖列表**，不再使用 `.mango-continue-grid` 多列宽卡网格。
2. **第 1 条**（`continue_reading[0]`）渲染为大卡：封面、标题、页码文案、粗进度条、单一主按钮「继续」→ `reader/{title_id}/{id}`。
3. **第 2 条及以后**渲染为紧凑列表行：小封面、标题、细进度条；整行可点 → 阅读器；无页码、无按钮。
4. 仅 1 条时只显示大卡，不渲染空列表容器。
5. 列表项总数（除大卡外）默认最多显示 **3** 行；超出时提供「展开更多 / 收起」；展开后显示剩余项（上限仍为 API 返回的最多 7 行列表）。
6. 进度在大卡与列表中均清晰可见，便于扫读。
7. 布局形态与「开始阅读 / 最近添加」的横向 poster rail **明显不同**。
8. 无数据时行为不变：整块 section 不渲染。
9. 本轮不改后端 limit/排序；不新增独立历史页；大卡不提供「打开」书详情按钮。

## Acceptance Criteria

- [ ] 首页有继续阅读数据时，区块为「1 大卡 + 可选列表」，不是多列宽卡网格。
- [ ] 大卡展示封面、标题、页码、进度条、「继续」；无「打开」按钮。
- [ ] 大卡「继续」进入对应 entry 阅读器 URL。
- [ ] 第 2+ 条为紧凑行；整行点击进入对应阅读器。
- [ ] 仅 1 条时只有大卡、无列表区。
- [ ] 列表默认最多 3 行；第 5 条及以后（即列表第 4 行起）默认隐藏，可通过展开看到；可收起回默认。
- [ ] 展开控件仅在列表项 > 3 时出现。
- [ ] 视觉上与 start/recent 的 poster rail 不同（非横向海报轨）。
- [ ] 相对旧宽卡网格，同等条数下垂直占用明显更低（定性验收即可）。
- [ ] i18n：新增展开/收起文案（若现有 key 不足）；中英文可用。
- [ ] 不改动 `api/home` 的 limit=8 与排序语义。

## Out of Scope

- 独立「阅读历史」页面
- 后端 `GetContinueReading` / `apiHome` limit 或排序变更
- 阅读器内进度 UI
- 大卡「打开」书详情
- 将继续阅读改成与 start/recent 同款 rail
- 列表行显示页码或次要按钮
